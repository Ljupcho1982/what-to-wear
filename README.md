# What to Wear — Што да облечам

A minimal app that tells you what to wear each morning, from real-time weather and
your own wardrobe. No photos, no cataloguing — a fast daily recommendation.

Built as an installable **PWA** and wrappable with **Capacitor** for Google Play,
matching the ParkinsonCare / Health Diary pipeline.

## What it does (v1)
- One-time onboarding: location (GPS or city search) + commute mode + wardrobe.
- **Today**: live weather (temp, feels-like, wind, rain %, UV) + a suggested outfit
  by category, with a **Not today** button that swaps any item for the next-best.
- **Wardrobe**: add / edit / delete items; toggle 🧺 to mark something in the laundry.
- **Settings**: notification time, °C/°F, commute mode, change location, reset.
- Daily local notification with the day's outfit; offline app shell via service worker.

## Architecture
| File | Role |
|---|---|
| `index.html` | UI shell + styles (dark/light, system-driven) |
| `app.js` | screens, storage (localStorage), CRUD, notifications |
| `recommend.js` | **pure** rule-based engine — no DOM/network, unit-tested |
| `weather.js` | Open-Meteo forecast + geocoding (free, no API key) |
| `recommend.test.mjs` | engine unit tests (`node --test`) |
| `sw.js`, `manifest.webmanifest`, `icons/` | PWA plumbing |

The recommendation engine always reasons in °C; °F is a display-only conversion.

### Recommendation rules
Base layer by feels-like temperature (>22 / 15–22 / 8–15 / 0–8 / <0 °C), then modifiers:
wind > threshold → windproof outer; rain > threshold → waterproof item (else "bring an
umbrella" flag); UV > 6 → sun accessory. Commute mode tightens the wind/rain thresholds
for walkers vs drivers. If no item matches the required warmth/tag it falls back to the
closest and flags "no ideal match". Inactive (laundry) items are never suggested.

## Run locally
```bash
# any static server works
python -m http.server 4323
# open http://localhost:4323
```

## Test the engine
```bash
node --test        # 11 tests, incl. the "never suggest non-waterproof on rainy days" check
```

## Regenerate icons
```bash
npm run icons      # dependency-free PNG generator → icons/
```

## Build the Android app (Capacitor → Google Play)
Reuses the same flow as ParkinsonCare. From this folder:
```bash
npm install                 # installs Capacitor CLI + core
npm run android:add         # assembles www/ and adds the native android/ project
npm run open:android        # opens Android Studio
```
After code changes: `npm run sync` (rebuilds `www/` and copies into the native project).

In Android Studio, build a signed **AAB**. Reuse your existing upload keystore
(`upload-keystore.jks`) and the **closed testing track / tester group** already set up
for ParkinsonCare. App id: `com.ljupcho.whattowear`.

> `www/`, `android/`, and `node_modules/` are git-ignored — `www/` is generated by
> `npm run build`, `android/` by `npx cap add android`.

## Not in v1 (stretch)
ML learning, outfit photos, sharing, multi-day trip packing, home-screen widget,
Macedonian-default UI. The engine already logs dismissed items
(`prefs.dismissedSuggestions`) as a seed for future lightweight weighting.
